# Feature Specification: 小老師進場與身份體驗 (Helper Onboarding)

**Feature Branch**: `003-helper-onboarding`
**Created**: 2026-06-25
**Status**: Draft
**Input**: walk-through 小老師端 Stop 2 ~ Stop 4 後識別的調整集合 —— 掃碼入口簡化、進房間儀式感、自我聲明印章、登記畫面身份視覺強化、換座號流程

**前置依賴**: `specs/001-little-teacher-helper/` 已實作完成。本 feature 在其上做擴充與修正，不取代原 spec。

**對應原則**（簡述，完整內容見 `specs/vision.md`）:

- **使用者是兒童** → 文案兒童化、不能在同學面前卡住、不要把學生當小孩
- **承諾裝置（commitment device）** → 自我聲明印章 + 身份常駐顯示，用視覺事實取代道德說教
- **鷹架不是黑盒（對學生）** → 錯誤一定要鋪好出路（自動降級到輸入框、連續失敗升級為找老師）
- **可逆性決定保護強度** → 換座號要重新入場（顯式行為）但不阻擋（用彈窗引導，不用權限攔截）

**路由地圖（本 feature 涉及）**:

```
/join                              ← 簡化版掃碼入口（US1）
/join/[code]                       ← 過場 + 自我聲明印章（US2）
/helper/[roomId]                   ← 任務清單（沿用既有，本 feature 不大改）
/helper/[roomId]/[taskId]          ← 登記畫面身份視覺強化（US3）
```

**「不該卡住」原則的擴充說明**：

本 feature 為 vision 第 2 節「鷹架不是黑盒」歸納了一條對小老師端通用的子原則：**「不讓學生在不知道下一步的狀況下卡住」**。具體應用：

- 相機 / 瀏覽器不支援 → 自動把焦點切到下方輸入框（不要讓他自己摸索）
- 連續失敗 3 次以上 → 升級為「去找老師」（不要繼續讓他試）
- 拒絕相機權限 → 同時告訴他「找老師」與「也可以直接打字」（兩條路都鋪好）

---

## User Scenarios & Testing _(mandatory)_

### User Story 1 - 掃碼入口簡化與錯誤兒童化 (Priority: P1)

`/join` 的「點此開啟相機掃描」中介按鈕拿掉，預設就是「相機圖示 + 開始掃描」單一狀態；按下後相機接管，**不出現「取消掃描」按鈕**（要切到手動輸入往下捲動即可）。所有錯誤訊息使用兒童語氣，並依情境提供具體出路。

**Why this priority**: vision 第 4 節「他不能在同學面前卡住」是入口最直接的考驗。中介按鈕讓使用者多按一下沒有收益；錯誤情境若用語生硬會讓孩子放棄。

**Independent Test**: 進入 `/join` → 第一眼看到「開始掃描」按鈕 → 按下掃描 → 拒絕相機權限 → 退回開始狀態 + 顯示「你還沒讓我看到相機，去找老師幫你打開吧。你也可以在下面直接打房間碼」+ 自動聚焦到下方輸入框；連續打錯 3 次房間碼 → 「試了好幾次都沒成功，去找老師看看吧」。

**Acceptance Scenarios**:

1. **Given** 老師打開 `/join`, **When** 系統渲染頁面, **Then** 預設狀態顯示「相機圖示 + 開始掃描按鈕」單一狀態（**不**先顯示「點此開啟相機」中介虛線框）
2. **Given** 使用者按「開始掃描」, **When** 相機接管掃描區, **Then** 掃描區內 **不顯示「取消掃描」按鈕**
3. **Given** 使用者按「開始掃描」並拒絕相機權限, **When** 系統偵測權限被拒, **Then** 自動退回「開始掃描」狀態 + 上方顯示「你還沒讓我看到相機，去找老師幫你打開吧。你也可以在下面直接打房間碼」 + 焦點自動跳到下方輸入框
4. **Given** 裝置沒可用相機或瀏覽器不支援 `getUserMedia`, **When** 使用者進入 `/join`, **Then** 相機區隱藏 + 焦點自動跳到下方輸入框
5. **Given** 使用者輸入錯誤房間碼, **When** 系統回覆「找不到」, **Then** 顯示「找不到這個房間耶 🔍 是不是少打了一個字？或是大小寫不一樣？」
6. **Given** 使用者連續輸入錯誤房間碼 3 次以上, **When** 系統累計失敗計數, **Then** 顯示「試了好幾次都沒成功，去找老師看看吧」（升級訊息，取代第一次的提示）
7. **Given** 老師已停用該房間（`room.isActive = false`）, **When** 使用者輸入正確房間碼, **Then** 顯示「這個房間老師暫時關起來了，問問老師怎麼回事吧」
8. **Given** 使用者完全沒網路, **When** 使用者進入 `/join`, **Then** 顯示「現在沒有網路喔，先去找一個有 WiFi 的地方再試試」
9. **Given** 使用者掃到非本系統的 QRCode（解析後不是房間 URL）, **When** 系統判斷, **Then** 顯示「這個碼好像不是老師給的喔，再對一次嗎？」
10. **Given** 房間碼解析後對應的 room 已啟用且網路正常, **When** 使用者進入 `/join/[code]`, **Then** 進入 US2 過場

### User Story 2 - 進房間的儀式感（過場 + 自我聲明印章） (Priority: P1)

`/join/[code]` 由「狀態頁」改造為「**過場頁**」：1.5–2 秒自動進入選座號（不可跳過）。歡迎畫面 icon 改為慶祝符號、班級名稱字級最大最醒目。選座號後出現「自我聲明印章」：全螢幕亮出「我是 [座號] 號 [姓名]」停 1.5 秒 + 輕量短促音效（可靜音），對應 vision 第 7 節「承諾裝置」設計哲學 —— 用視覺事實讓自我聲明具有實際的問責重量。

**Why this priority**: 進場是小老師心理上「成為小老師」的轉折，缺少儀式感會讓他覺得這只是一個普通的網頁；自我聲明印章是 vision「承諾裝置」設計的核心觸發點，沒做就等於放棄了問責設計。

**Independent Test**: 掃碼進入 `/join/[code]` → 看到「進來了！歡迎加入班級！」+ 大字班級名稱 → 1.5–2 秒後自動進入選座號（不可跳過）→ 選座號 → 全螢幕「我是 24 號 王小明」停 1.5 秒 + 印章音效 → 進入任務清單。

**Acceptance Scenarios**:

1. **Given** 使用者掃碼成功並抵達 `/join/[code]`, **When** 系統渲染歡迎畫面, **Then** 顯示順序為：慶祝 icon（候選：`lucide:party-popper` / `lucide:hand`）→ **大字班級名稱**（為畫面最顯眼元素，字級大於歡迎語）→ 「歡迎加入」輔助文字
2. **Given** 歡迎畫面已顯示, **When** 1.5–2 秒過去, **Then** 系統 MUST 自動進入選座號畫面，**不提供「跳過」或「進入」按鈕**（避免使用者下意識點過、失去儀式感）
3. **Given** 使用者選擇一個座號, **When** 系統確認選擇, **Then** **全螢幕**亮出「我是 [座號] 號 [姓名]」停 1.5 秒
4. **Given** 自我聲明印章顯示中, **When** 系統觸發回饋, **Then** 播放**輕量短促音效**（≤ 0.3 秒、非「咚」這類重音）+ 觸發 haptic 震動（如裝置支援）
5. **Given** 裝置音量為 0 / 靜音模式, **When** 自我聲明印章顯示, **Then** 系統不播放音效；haptic 仍然觸發
6. **Given** 自我聲明印章顯示中, **When** 使用者點擊任何位置, **Then** **不可跳過**（停滿 1.5 秒後自動進入任務清單）
7. **Given** 自我聲明印章結束, **When** 系統淡出, **Then** 進入 `/helper/[roomId]` 任務清單

### User Story 3 - 登記畫面身份視覺強化 (Priority: P1)

`/helper/[roomId]/[taskId]` 的「受指定 vs 未指定」狀態目前只有顏色差異的 banner，視覺重量不足以傳達身份重要性。本 US 重構：拿掉中間 banner、將右上 header 的座號 badge 移到**學生名單外框的正上方**並加上「登記者：」標籤；依「是否為指定登記者」分流視覺。同時移除「2 已繳 5 未繳」繳交統計 badge（這應該是給老師看的、非小老師職責）。

**Why this priority**: vision 第 7 節「承諾裝置」需要「系統記錄了是誰登的、老師看得到」這件事在介面上**持續可見**。把身份 badge 移到名單正上方等於小老師每登記一筆都直接看到自己的座號顯眼地存在著。

**互動一致性**: 「登記者：」badge 不論受指定 / 未指定 / 沒指定任何人三種情境都會顯示，差別在 badge 的視覺強度 + 是否帶星星 + 是否有小行文字。

**Independent Test**: 進入登記頁 → 名單外框正上方有大型「登記者：[座號]」badge → 受指定者：badge 加星星 + 進場 fade-in 動畫 + 「你是老師指定的登記者！」小行字；未指定者（有指定其他人）：badge 改換不同色（非警告色） + 「你不是被指定的小老師>\_<」；未指定者（沒指定任何人）：純色 badge 無小字；繳交統計 badge 已不存在。

**Acceptance Scenarios**:

1. **Given** 小老師進入任一登記頁, **When** 系統渲染 header, **Then** 右上的座號 badge **MUST 移除**（移到 AS 2 的位置）
2. **Given** 小老師檢視登記頁, **When** 系統渲染學生名單區塊, **Then** 名單外框**正上方**顯示「登記者：[座號]」格式的大型 badge（字級與視覺重量明顯，不可作為附屬資訊處理）
3. **Given** 任務指定了登記者且小老師座號吻合（受指定）, **When** 系統渲染 badge, **Then** badge 加上星星 icon（`lucide:star`）+ 進場 fade-in 動畫（**僅播 1 次**，非持續閃爍）+ 下方小行文字「你是老師指定的登記者！」+ 中間原本的「指定就是你」彩色 banner **MUST 移除**
4. **Given** 任務指定了登記者但小老師座號不吻合（未指定，他人受指定）, **When** 系統渲染 badge, **Then** badge 採與 AS 3 不同的顏色（**非警告色** —— 不是紅黃，建議灰藍系）+ 下方小行文字「你不是被指定的小老師>\_<」+ 中間原本的 banner **MUST 移除**
5. **Given** 任務沒有指定任何登記者（`assignedSeatNumber == null`）, **When** 系統渲染 badge, **Then** badge 採中性色（與 AS 4 又不同；建議純黑邊白底）+ **不顯示**下方小行文字 + 中間原本的 banner **MUST 移除**
6. **Given** 學生名單區塊頂部目前的「2 已繳 5 未繳」徽章, **When** 系統渲染, **Then** **MUST 移除**（小老師看了會被推動去催同學，違反角色分工 —— 催繳是老師 / 家長責任）
7. **Given** 繳交類型任務與成績類型任務, **When** 系統渲染 AS 2 的「登記者：」badge, **Then** 兩種類型**統一**顯示登記者 badge（不因類型差異而省略）

### User Story 4 - 換座號流程 (Priority: P2)

點「登記者：」badge 觸發換座號流程：彈窗詢問「想換座號嗎？需要重新進入房間喔」+「重新進入」按鈕。確認後清掉本機房間 cache 並跳回 `/join`；網路狀態的處理統一由 `/join` 接手（不在彈窗判斷網路）。**不提供「回到剛才的房間」連結**，避免讓「換座號」變得太容易（避免聯想到身份冒用）。

**Why this priority**: 選錯座號是真實情境，必須給出路；但出路必須是「顯式行為」（重新入場）而非「就地切換」，以維持身份選擇的儀式感與責任感。

**Independent Test**: 點登記者 badge → 彈窗 → 確認 → 跳回 `/join`；離線情境下流程相同，由 `/join` 顯示離線提示。

**Acceptance Scenarios**:

1. **Given** 小老師在登記頁, **When** 點「登記者：[座號]」badge, **Then** 系統彈出對話框「想換座號嗎？需要重新進入房間喔」+「重新進入」+「取消」按鈕
2. **Given** 使用者按「重新進入」, **When** 系統處理, **Then** MUST 呼叫 `clearRoom(roomId)`（清掉本機房間 / 座號 / 學生 / 任務 cache）→ `router.push('/join')`
3. **Given** 使用者按「取消」或關閉對話框, **When** 系統處理, **Then** 對話框關閉、回到原本登記頁、無任何資料變動
4. **Given** 使用者抵達 `/join` 後處於離線狀態, **When** `/join` 渲染, **Then** 由 `/join` 顯示離線提示「現在沒有網路喔，先去找一個有 WiFi 的地方再試試」（US1 AS 8）；**彈窗本身不判斷網路**
5. **Given** 本機曾經有 cache 過的房間, **When** 換座號流程清掉 cache 後抵達 `/join`, **Then** **MUST NOT** 提供「回到剛才的房間」這類捷徑連結

---

### Edge Cases

- 自我聲明印章顯示期間裝置電量極低 / 系統突然鎖屏：印章中斷，小老師可能未進入任務清單。實作時 useEffect 卸載要 cleanup 計時器，避免重新進入時印章卡住
- 房間碼有效但學生名單為空：選座號畫面要顯示「老師還沒有把學生名單建好，去找老師喔」之空白狀態，**不該讓小老師卡在無人可選**
- 受指定座號 24 是「已被軟刪除學生」（移除入學異動）：選座號清單應不顯示已移除學生，但若小老師硬選了不存在的座號，系統需 fallback（建議：顯示「這個座號不存在喔」並退回選座號）
- 連續失敗 3 次升級訊息出現後，使用者重新整理頁面：失敗計數應在頁面層 state，重整即重置；不需要持久化到 localStorage

## Requirements _(mandatory)_

### Functional Requirements

- **FR-060**: `/join` 頁面 MUST 預設顯示「相機圖示 + 開始掃描」單一狀態，**移除**「點此開啟相機」中介虛線框
- **FR-061**: 掃描狀態下 MUST NOT 顯示「取消掃描」按鈕（透過往下捲動切到手動輸入區）
- **FR-062**: 相機權限被拒 MUST 自動退回「開始掃描」狀態，並在上方顯示提示文字（兒童語氣）+ 自動把焦點切到下方輸入框
- **FR-063**: 裝置沒可用相機 / 瀏覽器不支援 `getUserMedia` MUST 隱藏相機區塊 + 自動聚焦下方輸入框
- **FR-064**: `/join` 的所有錯誤訊息 MUST 使用兒童語氣（具體文案見 AS 5, 7, 8, 9）
- **FR-065**: 輸入錯誤房間碼 MUST 計數，連續 3 次或以上 MUST 升級為「去找老師」訊息
- **FR-066**: `/join/[code]` MUST 為「過場頁」：歡迎畫面 1.5–2 秒後自動進入選座號，**不提供跳過按鈕**
- **FR-067**: 歡迎畫面 icon MUST 為慶祝符號（`lucide:party-popper` / `lucide:hand` 或同類）；班級名稱字級 MUST 為畫面最大、最醒目元素
- **FR-068**: 選座號後 MUST 顯示「自我聲明印章」：全螢幕亮出「我是 [座號] 號 [姓名]」停 1.5 秒；**不可跳過**
- **FR-069**: 自我聲明印章 MUST 播放輕量短促音效（≤ 0.3 秒）；裝置靜音時不播音效；MUST 觸發 haptic（若支援）
- **FR-070**: `/helper/[roomId]/[taskId]` 登記頁 header 的座號 badge MUST 移除；改為「登記者：[座號]」格式的大型 badge 顯示於學生名單外框正上方
- **FR-071**: 「登記者：」badge MUST 依「是否為指定登記者 / 是否有指定人」三種狀態展現不同視覺：
  - 受指定 → 加星星 + 進場 fade-in 動畫（僅播 1 次）+ 小行字「你是老師指定的登記者！」
  - 未指定（他人受指定） → 不同色（非警告色）+ 小行字「你不是被指定的小老師>\_<」
  - 沒指定任何人 → 中性色 + 不顯示小行字
- **FR-072**: `RecordForm.tsx` 中間的「指定就是你」彩色 banner MUST 移除（資訊已由 FR-071 的 badge 承載）
- **FR-073**: 學生名單區塊頂部的「N 已繳 / N 未繳」徽章 MUST 移除（違反角色分工，催繳責任在老師 / 家長）
- **FR-074**: 點「登記者：」badge MUST 觸發換座號彈窗（「想換座號嗎？需要重新進入房間喔」+「重新進入」+「取消」）
- **FR-075**: 「重新進入」MUST 呼叫 `clearRoom(roomId)` 後跳 `/join`；網路狀態判斷統一由 `/join` 接手
- **FR-076**: `/join` 與相關離線流程 MUST NOT 提供「回到剛才的房間」捷徑連結

### Non-Functional Requirements

- **NFR-008**: 進場過場、自我聲明印章的動畫流暢度 MUST 達 60fps（特別在中低階平板上）
- **NFR-009**: 自我聲明印章音效檔案大小 MUST < 30KB（避免拖累首次載入）
- **NFR-010**: 失敗計數 state MUST 為 in-memory（頁面層），重整即重置；不持久化到 localStorage

### Key Entities (delta only)

無新增 entity。本 feature 為純 UI / 流程層改動。

## Success Criteria _(mandatory)_

- **SC-015**: 90% 的小老師能在掃碼後 10 秒內進入到任務清單畫面（含過場 + 印章時間）
- **SC-016**: 拒絕相機權限後，使用者能在 10 秒內找到並使用手動輸入區（透過自動聚焦達成）
- **SC-017**: 第一次擔任小老師的學生，看完自我聲明印章後能說出「老師會看到我登的東西」這類概念性理解（vision 第 2 節「為什麼時刻」設計目標）
- **SC-018**: 程式碼搜尋 `RecordForm.tsx` 不再出現 `submittedCount` / `notSubmittedCount` 之渲染（FR-073 驗證）

## Assumptions

- 平板裝置支援 Vibration API（haptic 回饋）；不支援時音效已足夠
- 平板瀏覽器 Web Audio API 可正常播放短促音效；首次互動前可能因 autoplay 政策被靜音（自我聲明印章發生在使用者點過「選座號」之後，已通過 autoplay gesture 要求）
- 小老師選錯座號後重新入場的頻率不高（< 5%），不需特別優化此流程
